Week 1 — Foundation

Network Services

A network's purpose is to make resources available. These resources—known as network services—include applications and the data they provide.

💻
Client-Server Apps

A client requests data from a server. Examples: Web (HTTP), Email (SMTP/POP3/IMAP4), File Transfer (FTP), Remote Desktop.

📁
File & Print Services

File servers provide shared storage. Print services let multiple users share one printer, reducing hardware costs.

📞
Communication Services

Convergence delivers data, voice & video on one network. UC centralises management. Voice/video is delay-sensitive but loss-tolerant.

QoS (Quality of Service) — Admins configure priority rules so voice/video traffic gets bandwidth first. Bandwidth = total data transmission capacity of the network.
Week 1 — Topology

Controlling Network Access

Logical topology describes how software controls access. The two primary models are Peer-to-Peer (P2P) and Client-Server.

Peer-to-Peer (P2P)

No centralized control. Each OS manages its own resources. Computers communicate directly.

✓ Simple, cheap, no server needed
✗ Not scalable, poor security, local accounts only
Client-Server Model

Access controlled by a centralized directory database via a Network Operating System (NOS) like Windows Server.

Active Directory (AD) — centralized database of user accounts & security rules. A Global Account lets users log into any network computer.
✓ Scalable, centralized security & monitoring
✗ Complex setup, expensive hardware
Model Trade-off Comparison

Week 2 — Auto Configuration Added

DHCP & Subnet Masks

DHCP automates IP address assignment. Subnet masks define which part of an IP is the network vs. the host.

DHCP — Dynamic Host Configuration Protocol

Automatically assigns IP addresses to devices on a network. The DORA process:

D Discover — Client broadcasts "Is there a DHCP server?"
O Offer — Server responds with an available IP address
R Request — Client formally requests the offered IP
A Acknowledge — Server confirms & assigns the IP (with a lease time)
DHCP failure → APIPA assigns 169.254.x.x — local only, no internet access.

Week 2 — Network Layer

IPv4 Addressing

32-bit addresses written as four decimal octets (0–255). The network class determines the split between network and host portions.

Hosts per Class (Log Scale)
Class A

First octet = Network. Remaining 3 = Host. ~16.7M hosts. Range: 1–126.

Class B

First 2 octets = Network. ~65,000 hosts. Range: 128–191.

Class C

First 3 octets = Network. 254 hosts. Range: 192–223.

Class D & E

D (224–239) = Multicast. E (240–254) = Reserved for research.

Special & Reserved Addresses
255.255.255.255Broadcast — Sent to every device on the local network.
127.0.0.1Loopback — "This computer." Used for local testing (ping yourself).
169.254.x.xAPIPA — Self-assigned when DHCP fails. Local communication only.
0.0.0.0Default Route — Used in routing tables as the "catch-all" gateway of last resort.

NAT — Network Address Translation

Substitutes private IPs with a single public IP before sending traffic out. Conserves public IP addresses.

PAT — Port Address Translation

Extension of NAT. Assigns a unique TCP port to each ongoing session so returning traffic knows which internal device requested it. Allows many devices to share one public IP.

Week 3 — Layer 4

TCP vs. UDP

The Transport layer chooses how data is delivered — absolute reliability (TCP) or raw speed (UDP).

TCP — Transmission Control Protocol
Connection-Oriented & Reliable
  • 3-Way Handshake: SYN → SYN/ACK → ACK before any data
  • Sequencing: Numbers segments for ordered reassembly
  • Checksums: Verifies data integrity
  • Flow Control: Prevents overwhelming receiver
  • Use: Web, email, file transfer — anything needing guaranteed delivery
UDP — User Datagram Protocol
Connectionless & Fast
  • No handshake: Just fires data immediately
  • No guarantees: No error checking, sequencing, or ACKs
  • Low overhead: Only 4 fields in the header
  • Use: Streaming, VoIP, gaming — speed over reliability
TCP Three-Way Handshake

TCP establishes a reliable connection BEFORE sending payload data.

Week 3 — Layers 2 & 3

Network & Data Link Protocols

These protocols handle mapping, routing, error reporting, and address resolution across local and global networks.

IP — Internet Protocol
L3

Specifies how and where data should be delivered. Enables internetworking (traversing multiple LANs via routers).

Important: IP is unreliable and connectionless on its own — it relies on TCP for reliability.
ICMP — Internet Control Message Protocol
L3

The network's "reporter." Reports on success/failure of delivery — expired TTL, unreachable destinations, congestion.

Key limit: ICMP only reports errors to the sender. It does NOT correct them.
IGMP — Internet Group Management Protocol
L3

Manages multicast groups on IPv4 networks. Routers use IGMP to determine which devices want to receive a specific multicast stream (e.g., video broadcast).

ARP — Address Resolution Protocol
L2 / L3

Discovers the physical MAC address when only an IP is known. Broadcasts "Who has 192.168.1.5?" — that device responds. Results stored in an ARP Table.

Limitation: ARP broadcasts cannot cross routers — local network only.
TTL (Time to Live) — A counter in each IP packet, decremented by 1 at each router hop. When it reaches 0, the packet is discarded and ICMP sends a "TTL Exceeded" message back to the sender. Prevents packets from looping forever.

Week 4 — Electrical Uptime

Power Management

Power fluctuations cause hardware damage and intermittent network failures. Redundancy and conditioning maintain uptime.

Wall Power Anomalies vs. Online UPS Output
Surge

Momentary voltage spike (lightning strikes).

Noise

Rapid fluctuations caused by EMI.

Brownout/Sag

Voltage drop — equipment behaves erratically.

Blackout

Complete power loss.

UPS Types
  • Standby UPS — Monitors line; switches to battery on failure. Brief gap during switchover.
  • Online UPS — Equipment ALWAYS runs from battery; wall power just charges it. Perfectly clean output.
Generators

For long-term blackouts (diesel/natural gas). Best practice: pair a generator with an Online UPS. The UPS bridges the gap while the generator spins up (typically 10–30 seconds).

Week 5 — Transmission Basics

Analog vs. Digital Signaling

All network data travels as signals. Toggle between signal types to see the fundamental difference in how data is encoded.

Analog Properties
  • Amplitude — Signal strength (wave height)
  • Frequency — Cycles per second (Hz)
  • Wavelength — Distance between peaks
  • Phase — Position in the cycle relative to a reference
Drawback: Highly susceptible to noise — any voltage variation corrupts the signal.
Digital Properties
  • Binary — Only two valid states: 1 or 0
  • State 1 — Positive voltage present
  • State 0 — Zero voltage
Advantage: Minor signal distortions can be corrected — "is it high or low?" is easy to determine.
Data Modulation: A Modem (MOdulator/DEModulator) encodes digital data onto an analog carrier wave via FM (modifies frequency) or AM (modifies amplitude).
Week 5 — Sharing the Wire

Transmission & Multiplexing

Multiplexing allows multiple signals to share one medium by dividing it into logical subchannels. A Mux combines signals at the source; a Demux separates them at the destination.

Baseband

Digital signals use the entire medium as a single channel. Exclusive use. Example: standard Ethernet.

Broadband

Multiple transmissions share a single physical medium via multiplexing. Example: Cable TV, DSL internet.

Types of Multiplexing

TDM — Time Division

Divides medium into time slots. Each device takes strict, equal turns transmitting.

Statistical Mux — Smart TDM

Assigns time slots dynamically based on priority and demand. More efficient than standard TDM.

FDM — Frequency Division

Assigns unique frequency bands to each subchannel. Signals travel simultaneously (DSL, Cellular).

WDM — Wavelength Division Fiber Only

Carries multiple light signals simultaneously using different colors (wavelengths) of light over fiber optic cable.

Throughput = the actual amount of data transmitted per unit time (payload rate). Bandwidth = the physical range of frequencies a medium can transmit (its capacity ceiling). In networking, both are expressed as bit rates (Mbps, Gbps).
Week 5 — Physical Layer

Copper Cabling — Twisted Pair

The most common network cabling. Wires are twisted together to reduce cross talk (signal bleed between adjacent pairs). Max segment: 100 meters.

Twist Ratio: More twists per meter = better cross talk resistance, higher cost, but MORE attenuation (signal loss over distance). Engineering trade-off.
UTP — Unshielded Twisted Pair
  • Wire pairs directly in plastic sheath
  • ✓ Less expensive, highly flexible
  • ✗ Less resistant to EMI/noise
STP — Shielded Twisted Pair
  • Metallic foil/braid surrounds the pairs
  • ✓ Excellent EMI resistance
  • ✗ Expensive; requires proper grounding

RJ-45 Cable Pinouts

Straight-through

Both ends wired identically (T568A–T568A or T568B–T568B). Standard connection: computer to switch.

Crossover

Transmit/receive wires reversed on one end. Used for switch-to-switch or computer-to-computer.

Rollover (Console)

All wires completely mirrored. Used to connect a PC to a router/switch console port for configuration.

Week 5 — Coaxial Cable Added

Coaxial & Other Copper Media

Coaxial cable predates twisted pair in networking but is still widely used in cable TV, broadband internet connections, and some legacy network installations.

Coaxial Cable Structure
Core — Solid copper conductor (carries signal)
Dielectric — Insulating layer separating core from shield
Braided Shield — Metal mesh protecting against EMI
Outer Jacket — PVC protective sheath
RG-58 (Thin Coax)

Used in legacy 10Base2 Ethernet ("Thinnet"). Max segment 185m. Rarely used today.

RG-8 (Thick Coax)

Used in legacy 10Base5 Ethernet ("Thicknet"). Max segment 500m. Now obsolete in networking.

RG-6 / RG-59 (Modern)

Used for cable TV (CATV), cable modem broadband (DOCSIS standard), and satellite connections.

Legacy Alert: Coaxial Ethernet (Bus Topology) has a critical single point of failure — one break anywhere in the cable brings down the entire network segment. This is why star topology with twisted pair replaced it.

Week 6 — RF Fundamentals

Wireless Signals & Spectrum

WLANs transmit data via Radio Frequency (RF) waves. They use the same Layer 3+ protocols as wired networks, but the open-air medium introduces unique challenges.

Frequency Bands
2.4 GHz BandOlder
Range: 2.4 to 2.4835 GHz — 11 unlicensed channels
Congested: Shared with microwaves, baby monitors, Bluetooth
5 GHz BandNewer
24 unlicensed bands, each 20 MHz wide — far less congestion
Regulated by FCC (USA) and ITU (International). Spectrum runs 9 KHz → 300 GHz.
Antenna Types Added
Omnidirectional Antenna

Broadcasts signal in all directions equally (360°). Used in home Wi-Fi routers and WAPs for general coverage.

Unidirectional (Directional) Antenna

Focuses signal in one specific direction. Used for point-to-point links (building-to-building). Longer range, less interference.

Signal Propagation Behaviors

➤➤
➤➤
Reflection

Signal bounces back toward its source when hitting a dense, smooth surface (metal walls, elevator doors, windows).

Multipath Signals: Reflection/diffraction/scattering cause the same signal to arrive via multiple paths at slightly different times — causing data errors and degraded performance.

Week 6 — IEEE 802.11

Wi-Fi Standards

All 802.11 standards are half-duplex — a device can either transmit OR receive, never both simultaneously. Throughput has grown exponentially across generations.

Half-Duplex: Because all devices share the air medium, only one can transmit at a time. This is why collision avoidance (CSMA/CA) is necessary and why real-world throughput is always well below the theoretical maximum.

Week 6 — Access & Protection

Access Methods & Security

Wireless devices can't transmit and receive simultaneously — they use collision avoidance. The open-air medium also demands strong encryption.

CSMA/CA — Collision Avoidance

Carrier Sense Multiple Access with Collision Avoidance:

  1. Node listens to check if airways are clear
  2. If clear, transmits data
  3. Waits for an ACK packet to confirm receipt
  4. If no ACK, assumes collision — waits random time and retries
Overhead: The ACK requirement means real-world throughput is always well below theoretical maximum.
Association & Scanning
Active Device sends a Probe frame broadcasting "Any networks here?"
Passive Device listens for Beacon frames periodically broadcast by the WAP containing the SSID
Roaming — Moving between WAPs while maintaining connection (ESS). Risk: Rogue APs mimicking legitimate SSIDs to intercept traffic (Evil Twin attack).
Default Security: 802.11 provides NO security by default. Only an SSID is required to connect. Explicit security measures are essential.
Authentication

Matching client credentials against a database to verify identity before granting network access.

MAC Filtering

WAP only authenticates devices with MAC addresses on an approved allowlist. Not foolproof — MACs can be spoofed.

WPA2 Encryption

Strong AES encryption standard. Replaces weaker WEP/WPA. Enterprise mode adds a RADIUS server for individual user authentication (username + password per user).
